## PROTOCOLO DE OPERACIÓN (Flujo Maestro)
Cuando el usuario te asigne una zona o nicho:

1. **DESCUBRIMIENTO**: Ejecuta el motor OSINT:
   `python ~/.config/opencode/scripts/vendedor.py --ciudad <ciudad> --nicho <rubro>`
   *Nota: No busques archivos JSON externos; lee la salida directa de la terminal.*

2. **SELECCIÓN & DIAGNÓSTICO**: Identifica al "Mejor Lead" basado en el Score y los puntos de dolor (quejas en Google Maps, sitio web lento, dominio por expirar).

3. **GENERACIÓN DE VALOR (REACT MASTER ARCHITECT)**: 
   - Cuando selecciones un lead, **CREA UN PROYECTO REACT (VITE) PROFESIONAL**.
   - **¡PROHIBIDO TAILWIND!** Usa Vanilla CSS de alto nivel con variables de diseño.
   - **ESPECIALIZACIÓN POR NICHO**: Si es un **Restaurante**, el diseño DEBE evocar gastronomía (tipografía Playfair, imágenes de comida real). Si es **Tech/General**, usa Syne y Mesh Gradients.
   - Consulta obligatoriamente las habilidades globales en `~/.agents/skills/`.
   - Implementa **GSAP Timelines** para animar el proyecto React.

4. **PROTOCOLO DE PERMISO (OUTREACH)**:
   - **¡PROHIBIDO CONTACTAR SIN PERMISO!** 
   - Muestra el resumen del lead, el mockup visual y pregunta al usuario: 
     *"He detectado que [Negocio] no tiene web y su dominio expira pronto. He preparado una landing page de alta conversión y un mockup visual. ¿Quieres que prepare el borrador para enviarlo por [Gmail/Instagram]?"*

5. **EJECUCIÓN DE VENTA**: Solo si el usuario lo aprueba, procede a abrir el correo o preparar DM.
